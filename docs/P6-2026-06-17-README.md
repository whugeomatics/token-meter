# P6 Team Trend Intelligence

## Goal

P6 turns the existing Team dashboard from usage reporting into usage change explanation.

The product goal is to help a team understand token usage trends: what changed this week or month, which users/devices/models/tools caused the change, and whether upload health affected the numbers.

P6 is not a gateway phase. It does not add a request proxy, local model gateway, provider routing, billing, pricing, budgets, cloud sync, or new sensitive data collection.

## Inputs

P6 must use data that already exists in the P5 model:

- canonical usage events for Codex and Claude Code;
- Local and Team period comparison windows;
- Team upload health data;
- existing dimensions: team, user, device, tool, model, date.

## Scope

- Add a `What changed?` view for Team reports.
- Explain total token and net usage deltas by user, device, model, and tool.
- Improve week-over-week and month-to-date comparisons.
- Surface trend signals for token spikes, cache hit drops, reasoning ratio changes, duplicate uploads, rejected uploads, and stale devices.
- Add a copyable Markdown summary for weekly and monthly reporting.
- Improve empty-state guidance based on whether other filters have data.

## Non-Goals

- No local model gateway.
- No request forwarding proxy.
- No provider adapters or provider auto-routing.
- No pricing, cost estimation, budgets, billing, or alerting platform.
- No cloud account, cloud sync, or login system.
- No prompt, response, raw JSONL, raw API body, transcript text, user input, or model output collection.

## Success Criteria

- A team owner can see the largest positive and negative token changes for the selected period.
- The dashboard can explain the main drivers by user, device, model, and tool.
- Upload health trends are visible when they can affect report trust.
- Markdown summary reflects the current filters and current period.
- P1/P2 Local report compatibility, P3 Team report compatibility, and P5 token formulas remain intact.
