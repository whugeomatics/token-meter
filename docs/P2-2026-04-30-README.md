## 当前状态

P2 已实现并通过用户真实 Windows 终端验收。

2026-04-30 mac 修复追加：

- dashboard 服务启动前自动执行一次 Codex ingestion，避免页面只读取空 SQLite；Local 页面加载、切换时间范围和 Refresh 也会先执行一次增量 ingestion，再读取 `/api/report`。
- 默认 SQLite 存储从单文件调整为月分片目录 `~/.token-meter/sqlite/token-meter-YYYY-MM.sqlite`。
- `--db=<path>` 指向 `.sqlite` / `.db` 文件时继续使用单文件，兼容既有脚本和临时测试。
- 新增 mac smoke test：`sh scripts/P2-2026-04-30-smoke-test.sh`。
- 没有 `token_count` 的旧 Codex session 只记录 source checkpoint，不生成 token usage，避免推算虚假数据。

本阶段最终约束：

- 不实现 Claude Code。
- 不实现 Cursor。
- 不实现本地模型网关。
- 不存 prompt/response 正文。
- 不改变 P1 `/api/report` contract，除非先更新 contract 和验收标准。

## 实现入口

后续 Backend Agent 应优先执行：

## 架构整理

P2 验收后补充了结构性整理：

- 前端页面迁移到 `src/main/resources/static/index.html`。
- SQLite SQL 迁移到 `src/main/resources/db/schema-v1.sql`。
- Java 通过 `org.xerial:sqlite-jdbc` 和 JDBC API 访问 SQLite。
- 后端拆分为入口、配置、HTTP、ingestion、repository、report、domain 和 utility 文件。
