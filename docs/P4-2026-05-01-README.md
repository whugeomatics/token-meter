# P4 2026-05-01 Claude Code Team Usage Collection

P4 的目标是在 P3 Codex team collection 已跑通的基础上，让 local dashboard 和 teammate collector 支持 Claude Code usage metadata，使本地和团队统计都能从单一 Codex 扩展到至少两个真实 AI CLI 工具。

本阶段只做 Claude Code 使用上报，不做 Cursor、本地模型网关、多 provider 自动路由、登录、云同步或计费系统。

## MVP 边界

允许：

- teammate collector 默认同时采集 Codex 和 Claude Code；客户端无需传单独 Claude 开关。
- local dashboard 启动和 `/api/ingest` 同时采集 Codex 与 Claude Code，并通过 `tool` 维度区分。
- 默认读取用户主目录下的 Claude Code 本地项目日志：macOS/Linux 与 Windows 均使用 `<user.home>/.claude/projects/**/*.jsonl`。
- 优先使用 Claude Code OpenTelemetry usage/cost/tool activity metadata。
- 在本地统计场景中允许读取 Claude Code JSONL 的 `sessionId`、`timestamp`、`message.model`、`message.usage` metadata。
- Hook 或 transcript path 只作为受限补充来源，用于 session/tool 元数据，不读取 transcript 正文。
- 复用 P3 device token、team/user/device binding、`/api/team/ingest` 和 `/api/team/report`。
- 服务端继续按 team、user、device、tool、model、date 聚合。
- Dashboard 能区分 Codex 和 Claude Code。
- 缺失字段必须显式记录为 unknown、0 或 source_quality 标记，不允许伪造。

禁止：

- 不采集 prompt 正文。
- 不采集 response 正文。
- 不采集 raw API body。
- 不采集 raw transcript 正文。
- 不上传 Claude Code 原始日志或完整本地路径。
- 不解析、存储或上传 Claude Code JSONL 的 `message.content`。
- 不信任客户端自报 user/device 作为最终归属。
- 不实现 Cursor。
- 不实现本地模型网关。
- 不实现多 provider 自动路由。

## 核心数据流

```text
member laptop Claude Code local JSONL or telemetry source
  -> local app + team collector default collection
  -> normalized tool usage event batch
  -> existing /api/team/ingest
  -> device token binding lookup
  -> central SQLite monthly usage event shards
  -> /api/team/report with tool dimension
  -> team dashboard
```

## 采集策略

P4 首选 Claude Code OpenTelemetry 输出，因为它是官方 usage monitoring 入口，能提供 usage/cost/tool activity 级别的 metadata。Collector 不直接读取 Claude Code prompt、response 或 raw transcript。

Hook 只在 OTel 无法覆盖必要 metadata 时作为补充。Hook payload 中可能出现 `transcript_path`、`session_id`、tool 相关信息；P4 只允许读取路径本身或 session metadata，不读取 transcript 文件正文。

## 与 P3 的关系

P4 不新增第二套 server ingestion API。Claude Code usage event 应转换为 P3 team ingestion 可接收的通用 team usage event，并通过 `tool=claude-code` 区分来源。

P4 可以扩展 team report 的 response 字段以增加 `tools`、tool filter 和 tool comparison，但不得破坏 P3 既有字段。
