# P5 2026-05-24 Unified CLI Usage Metrics

P5 的目标是在 P4 已经跑通 Codex 和 Claude Code local/team usage collection 后，把两种工具的统计字段、缺失字段处理和 report 派生公式统一成稳定的 CLI usage metrics schema。

本阶段允许做少量代码调整，但只限于让现有 Codex 和 Claude Code 实现贴合 unified metrics contract。P5 不接入新的 CLI 工具，不实现本地模型网关，不做 provider 自动路由、费用估算、预算或告警。

## MVP 边界

允许：

- 定义 canonical usage event 事实字段。
- 必要时补齐 event/DB/API 中缺失的事实字段。
- 统一 Local 和 Team report 的 token 字段含义。
- 统一 Codex 和 Claude Code 的 field mapping。
- 统一 `net_input_tokens`、`net_total_tokens` 和 `cache_rate` 的 report 层计算公式。
- 统一缺失字段 fallback 和无效 usage 过滤规则。
- 更新 dashboard 展示文案或字段使用，使同名指标含义一致。
- 为未来 CLI 预留 source mapping 模式和可选 `provider` 字段。

禁止：

- 不接入 Cursor。
- 不接入 Gemini CLI。
- 不实现本地模型网关。
- 不实现多 provider 自动路由。
- 不引入登录、云同步、计费、预算或告警。
- 不把 `net_*` 或 `cache_rate` 作为新的历史事实写入 DB。
- 不保存 prompt 正文、response 正文、raw JSONL、raw API body、transcript 正文、admin token、device token 明文或 token hash。

## 核心原则

P5 将字段分成两类：

- 事实字段：来源工具明确上报或 collector 可以无正文推导的事件事实，允许进入 canonical event、payload 和 DB。
- 派生指标：由 report 层按统一公式计算的可比较指标，不作为新的历史事实写入 DB。

事实字段包括：

```text
tool
provider
source_kind
source_quality
model
session_id
timestamp
input_tokens
cached_input_tokens
output_tokens
reasoning_output_tokens
total_tokens
```

派生指标包括：

```text
net_input_tokens
net_total_tokens
cache_rate
```

## 当前阶段完成标准

- Codex 和 Claude Code 都能映射到同一套 canonical usage event 字段。
- Local `/api/report` 和 Team `/api/team/report` 对同名 token 字段使用同一语义。
- summary、daily、models、sessions、tools 和 comparison 使用同一派生指标公式。
- integration 文档能说明 Codex 和 Claude Code 的原始字段到 canonical 字段的映射。
- 验证命令通过或在 acceptance 中记录明确限制。
