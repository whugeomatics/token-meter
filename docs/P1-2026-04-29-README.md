## 命名约定

所有阶段性文档必须使用以下格式：

```text
P<阶段号>-YYYY-MM-DD-<主题>.md
```

例子：

- `P1-2026-04-29-roadmap.md`
- `P2-2026-05-01-codex-log-research.md`
- `P4-2026-05-01-claude-code-usage-event.md`

`README` 也必须带阶段和日期，因为它代表该阶段的索引快照。

## 当前原则

1. 先做只读统计，不改 Codex，不代理请求。
2. 先验证 Codex 日志能稳定支撑 dashboard。
3. 等 Codex dashboard 有价值后，先补齐 Claude Code 等 CLI 工具统计，再进入本地模型网关。
4. 所有阶段都要有设计文档、开发任务、验收报告。
5. P1 已补齐需求、调研计划、API contract、dashboard 信息架构、任务拆分、后端原型和 dashboard 首页。
6. 进入 P2 前必须在用户本机补跑 Maven package 与 smoke test，并把结果写回 acceptance 文档。
