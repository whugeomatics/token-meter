# P4 2026-05-01 Claude Code Team Usage Collection Design

## 目标

P4 在 P3 Codex teammate collector 的基础上增加 Claude Code usage collection。完成后，Local 和 Team dashboard 能在同一套 report 体验里同时展示 Codex 和 Claude Code 的用量。

## 背景

P3 已经完成：

- device token 绑定 team/user/device。
- teammate collector 分发和周期上报。
- `/api/team/ingest` 幂等写入。
- `/api/team/report` team/user/device/model/date 聚合。
- Local/Team 日、周、月自然周期对比。

P4 不新增第二套服务端采集链路。Claude Code 的差异应收敛在 local ingestion、collector source adapter 和 usage event normalization 中。

## 来源选择

Claude Code 官方支持 usage monitoring。P4 首选 OpenTelemetry usage/cost/tool activity metadata，因为它最接近产品需要的 usage 上报能力，并且可以避开 prompt/response 正文。

Hook 可以提供 session/tool 相关 metadata，但 Hook payload 可能靠近用户输入、tool input 或 transcript，因此 P4 只把 Hook 作为补充来源，不读取 transcript 正文，也不上传 raw hook payload。

## 数据流

```text
Claude Code local JSONL or telemetry metadata
  -> ClaudeCodeUsageSource
  -> local usage event or TeamUsagePayload tool=claude-code
  -> existing /api/team/ingest
  -> existing team SQLite shards
  -> /api/team/report?tool=claude-code
  -> Team dashboard
```

## Collector 设计

默认 team collector 一次运行同时采集 Codex 和 Claude Code。teammate 不需要传 Claude 专用开关；`--collect-claude-code` 仅作为旧脚本兼容入口保留。

collector teammate 配置优先级固定为：

```text
CLI 参数 > ~/.token-meter/collector.env > 系统环境变量
```

macOS/Linux 默认读取 `~/.token-meter/collector.env`。Windows 默认读取 `%USERPROFILE%\.token-meter\collector.env`，并兼容旧的 `collector.env.cmd`。server 端不读取 teammate `.env`。

新增 source adapter：

```text
ClaudeCodeUsageSource
  -> reads local JSONL usage metadata, OTel metadata export, hook metadata fixture, or future source
  -> validates privacy boundary
  -> maps record to local usage event or TeamUsageEvent
  -> sets tool=claude-code
```

P4 不引入本地 checkpoint DB。周期运行可以重新扫描最近窗口，重复由服务端 `event_key` 去重。

## Server 设计

P4 复用 P3 server endpoint：

```text
POST /api/team/ingest
GET /api/team/report
```

服务端需要确认：

- `tool=claude-code` 可以写入。
- report 聚合按 tool 维度分桶。
- `tool` query filter 不破坏旧查询。
- period comparison 支持 tool delta。

## Dashboard 设计

Local 和 Team 页面增加 tool 维度：

- All Tools。
- Codex。
- Claude Code。
- Tool ranking。
- Period Comparison 中展示 Codex vs Claude Code 的 token delta。
- Models/Sessions 等表格中展示 tool 来源。

页面不显示来源 raw payload，不显示 transcript path 明文。

## 隐私和安全

P4 继续遵守 P3 token 和隐私规则：

- 服务端归属来自 device token binding。
- 不信任客户端自报 user/device。
- device token 不写入日志。
- admin token 不写入日志。

Claude Code 专属禁止项：

- 不读取 transcript 正文。
- 不上传 prompt。
- 不上传 response。
- 不上传 raw API body。
- 不上传 tool input/output payload。

## 非目标

- 不接入 Cursor。
- 不实现本地模型网关。
- 不实现多 provider 自动路由。
- 不做计费系统。
- 不做云同步。
- 不要求 Claude Code 常驻 collector。

## 验证策略

- 使用脱敏 Claude Code telemetry fixture 驱动 collector 单测。
- 使用 Claude Code local JSONL fixture 验证本地 `message.id` 去重。
- 使用 server ingestion fixture 验证 `tool=claude-code` 写入和幂等。
- 使用 team report fixture 验证 `tool` filter、`tools` response 和 period comparison。
- 使用隐私 fixture 确认 prompt/response/raw body/transcript 字段被忽略。
- 使用 collector `.env` fixture 验证配置优先级。
