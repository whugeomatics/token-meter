## 建议开发顺序

1. Server foundation：device token schema、token hash lookup、team usage event schema。
2. Ingestion API：payload validation、token binding、idempotent insert、response summary。
3. Collector：复用 core parser，新增轻量 upload CLI，不依赖数据库。
4. Report：新增 `/api/team/report`，返回团队总览、用户明细、设备明细、模型聚合和趋势，不破坏 P1/P2 `/api/report`。
5. Dashboard：增加团队视角，默认看总览，并能查看每个用户的使用情况。
6. Smoke test：覆盖 token 绑定、重复上报、隐私字段、聚合结果。

## 实现记录

- 后端已按职责拆为 `app`、`http`、`ingestion`、`domain`、`report`、`store`、`util` package。
- 前端已拆分为 `index.html`、`app.css`、`app.js`。
- device token 以 SHA-256 hash 用于鉴权查找，同时 registry 保存可恢复 token 供管理员后续复制。
- `--create-device-token` 用于生成并绑定设备 token。
- admin token binding 页面只展示 mask 后的 token，支持复制和删除，不支持编辑。
- `--register-device-token` 用于预置设备 token 绑定。
- `--admin-token` 或 `TOKEN_METER_ADMIN_TOKEN` 用于开启管理员页面。
- `--collect-team` 用于 collector 本地采集并主动上报。
- collector 不依赖本机数据库；collector 不启动 HTTP server，不占用 server 端口。
- `scripts/P3-2026-05-01-run-collector.sh` 是独立 collector 启动脚本。
- `scripts/P3-2026-05-01-package-collector.sh` 可生成 teammate 分发目录，目录内只需要 jar 和 sh。
- `scripts/P3-2026-05-01-install-collector-service.sh` 可把 collector 安装成 mac LaunchAgent 周期执行。
- `scripts/P3-2026-05-01-uninstall-collector-service.sh` 可卸载 LaunchAgent。
- `--team-report` 用于 CLI 查看团队 report。
- `--team-ingest-file` 仅用于 smoke test 和本地排障，产品入口仍是 `POST /api/team/ingest`。
- collector 直接扫描最近窗口内的 Codex sessions，不依赖本地 SQLite checkpoint；网络失败时可重跑 collector，服务端依赖 `event_key` 幂等去重。
- team token 绑定默认保存在 `token-meter-team-registry.sqlite`，team usage event 默认按月保存在 `token-meter-team-YYYY-MM.sqlite`。
- collector 默认单 batch 最多 500 条 event，可用 `--batch-size` 调整。
- 管理员使用指南记录在 `docs/guides/P3-2026-05-01-admin-usage-guide.md`。

## 多 Agent 分工建议

Backend Agent：

- 负责 server ingestion API、device token、SQLite schema、team report。

Collector Agent：

- 负责 collector CLI、最近窗口扫描和上传协议。
- 不引入本地数据库、HTTP server、dashboard 静态资源或 app CRUD/API/report/admin 实现。

Frontend Agent：

- 负责团队视角 dashboard，只消费后端 contract。
- 必须展示团队总用量和每个用户的用量，不在前端重新聚合原始 event。

Review Agent：

- 重点检查身份归属、幂等、重复统计、隐私泄露。
- 设计 review 已记录到 `docs/reviews/P3-2026-04-30-codex-team-collection-design-review.md`。

Acceptance Agent：

- 按 `docs/acceptance/P3-2026-04-30-codex-team-collection.md` 执行验收并记录结果。
