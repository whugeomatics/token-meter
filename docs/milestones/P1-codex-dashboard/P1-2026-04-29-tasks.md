## Research Agent

任务：

- 验证 Codex session JSONL 路径。
- 验证 token usage 字段。
- 验证 model 归因方式。
- 验证 timestamp 和本地日期转换。

产出：

- 更新 `docs/research/P1-2026-04-29-codex-log-research.md`。

验收：

- 能给 Backend Agent 提供明确字段路径和缺失字段策略。

## Backend Agent

任务：

- [x] 实现只读 JSONL 扫描。
- [x] 提取 token usage metadata。
- [x] 按 date、model、session 聚合。
- [x] 实现 `/api/report`。
- [x] 实现命令行自检模式 `--report`。

约束：

- 不修改 `.codex` 文件。
- 不记录 prompt 或 response 正文。
- 不实现 SQLite。
- 不实现本地网关。

验收：

- [x] `/api/report?days=7` 返回 contract 规定结构。
- [x] summary 等于 daily 聚合结果。
- [x] 重复读取同一日志不会在单次响应内重复计数。

实现记录：

- 见 `docs/milestones/P1-codex-dashboard/P1-2026-04-29-backend-prototype.md`。

## Frontend Agent

任务：

- [x] 实现 dashboard 首页。
- [x] 实现时间窗口切换。
- [x] 展示 summary、daily、models、sessions。
- [x] 实现空态和错误态。

约束：

- 不重新计算核心统计口径。
- 不做营销页。
- 不做登录或云同步。

验收：

- [x] 页面字段与 API contract 一致。
- [x] 无数据时页面可用。
- [x] API 错误时页面可读。

实现记录：

- `src/main/java/local/token/meter/TokenMeterApp.java` 提供 `/` 和 `/index.html`。
- 页面通过 `/api/report` 读取数据。
- 当前环境中本地 HTTP smoke test 被沙箱审批阻断，需在用户本机补跑。

## P1 完成定义

P1 完成必须同时满足：

- Codex usage dashboard 可本地启动。
- 真实 Codex 日志可读取。
- API contract 与页面展示一致。
- 隐私边界未被破坏。
- review 和 acceptance 文档已写入。
- AGENTS.md 如有新约定已更新。
