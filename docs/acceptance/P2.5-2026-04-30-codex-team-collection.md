- 时区：Asia/Shanghai

- 构建命令：`mvn -DskipTests package`

- Server 存储：临时 SQLite 目录或文件
- 验收日期：2026-04-30

## 验收项

### 1. 文档和阶段边界

结果：通过

检查：

- 文档明确 collector/app jar 分离，collector 不依赖 SQLite 或 dashboard 代码。

### 2. Device Token 归属

结果：通过

检查：

- 服务端可通过 device token 查到 `team_id`、`user_id`、`device_id`。
- 服务端使用 token hash 做鉴权查找。
- 服务端 registry 保存可恢复 token，以支持管理员后续复制；registry SQLite 按敏感数据处理。
- 服务端记录每次 collector 上报的 `team_id`、`user_id`、`device_id`、`upload_date`、`upload_time`、event 数量和处理结果。
- 服务端可通过 `--create-device-token` 生成并绑定明确使用人和设备的 token。
- 未知 token 返回 `401`。
- 禁用 token 返回 `403`。
- 客户端自报 `client_user_id` 与 token 绑定不一致时返回 `409`。
- usage event 最终归属来自 token 绑定，而不是客户端 payload。

### 2.1 管理员分配 Token

结果：通过

检查：

- 未配置 admin token 时管理页面不可用。
- 未登录管理员访问 `/admin.html` 被拒绝。
- 管理员可通过 `/admin-login.html` 登录。
- 登录后可以查看 device token 绑定列表。
- 登录后可以创建 teammate device token。
- 管理列表只展示 mask 后的 device token，不直接展示完整 token 或 hash。
- 管理员可点击复制按钮获取完整 device token。
- 管理员可删除 device token binding。
- 管理页面不允许编辑 token。
- admin token 不写入日志、DB、HTML、JS。
- 文档解释 Team 页面为 0 的常见原因：只创建 token 未上报、server-url 指错、日期范围不包含数据、身份字段与 token 绑定不一致、session 无 token usage。

### 3. Collector 本地解析

结果：通过

检查：

- collector 只读本机 Codex sessions。
- collector 只读取 token usage 相关字段。
- collector 能从 fixture 生成标准化 usage event。
- 没有 `token_count` 的 session 不生成 usage event。
- collector 不上传本机完整 source path。

### 4. 隐私

结果：通过

检查：

- 上传 payload 不包含 prompt。
- 上传 payload 不包含 response。
- 上传 payload 不包含 raw JSONL。
- 错误日志不包含 token 明文。
- 错误日志不包含原始 JSONL 行内容。

### 5. Server Ingestion

结果：通过

检查：

- `POST /api/team/ingest` 可接收合法 batch。
- 返回 `accepted`、`duplicate`、`rejected`。
- 非法 payload 返回 `400`。
- 重复 `event_key` 不重复统计。
- 写入事件包含 `team_id`、`user_id`、`device_id`、`received_at`。
- 上报日志和返回信息包含非敏感的 `upload_time`、上报人、设备、日期窗口信息，不包含 token。
- team usage event 写入 `agent-dashboard-team-YYYY-MM.sqlite` 月度分片。
- token 绑定写入 `agent-dashboard-team-registry.sqlite`。
- 单 batch 超过上限时拒绝，collector 默认按最多 500 条分批。

### 6. Retry 和 Idempotency

结果：通过

检查：

- collector 默认读取 `~/.codex/sessions`，日常使用无需传 `--sessions-dir`。
- collector 不依赖本地数据库，也不创建 checkpoint SQLite。
- collector 不启动 HTTP server，不占用 server 端口。
- collector 不初始化服务端 team registry。
- collector 可打包成只包含 jar 和 sh 的 teammate 分发目录。
- collector jar 不包含 dashboard HTTP/API、report、store、admin、static assets 或 SQLite driver。
- mac launchd collector 服务安装脚本存在，并使用周期拉起而不是 Java 常驻。
- 网络失败或 5xx 后，下次周期运行会重新扫描最近窗口并重试上报。
- 重试同一 batch 不造成重复统计。
- 400、401、403、409 不进入无限重试。

### 7. Team Report

结果：通过

检查：

- `GET /api/team/report` 返回团队总览。
- `GET /api/team/report` 返回每个用户的使用明细。
- 中央库可按 team 聚合。
- 中央库可按 user 聚合。
- 中央库可按 device 聚合。
- 中央库可按 model 聚合。
- Team Models 明细可按 `date + user_id + model` 聚合，包含 session 时间窗口。
- 中央库可按 date 聚合。
- P1/P2 `/api/report` contract 未被破坏。

### 8. Dashboard Team View

结果：通过

检查：

- 页面有明确的 Local 和 Team 两个 Tab。
- Local Tab 只展示本机 `/api/report`。
- Team Tab 只展示团队 `/api/team/report`。
- 页面可看到团队总 token、input token、output token、session、活跃用户、活跃设备。
- 页面可按 `team_id` 筛选不同团队。
- 页面可看到每个用户的 token、session、设备数、最近上报时间。
- 页面可区分同一用户下的不同设备。
- Team Models 表格展示 Date、User、Model、token、session、Started、Ended。
- Team Models 表格默认按日期倒序展示，并支持点击表头排序。
- Active 时间按 session 内相邻 usage event 间隔累加，超过 30 分钟的空闲间隔不计入，避免跨天复用 session 产生异常长活跃时间。
- Last Seen、Started、Ended 使用当前 report timezone 展示，不直接显示 UTC `Z`。
- Uploads 表格展示 team/user/device/upload date/upload time/event count/accepted/duplicate/rejected/status。
- 页面可查看团队按天趋势。
- 页面可查看用户筛选后的按天趋势或等价用户明细。
- 页面不展示 prompt、response、raw JSONL、完整 device token 或 token hash。
