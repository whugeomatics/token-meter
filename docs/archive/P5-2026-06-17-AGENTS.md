## 2. P5 Scope

P5: Unified CLI Usage Metrics.

P5 unifies Codex and Claude Code usage into one canonical CLI usage metrics model. It covers canonical usage facts, fallback rules, Local/Team derived report formulas, dashboard metric naming, source mapping documents, and schema shape for future CLI tools.

P5 does not add a new CLI source. It does not implement Cursor, Gemini CLI, a local model gateway, provider auto-routing, pricing, budgets, alerts, login, billing, or cloud sync.


P5 derived report metrics:

```text
net_input_tokens = max(input_tokens - cached_input_tokens, 0)
net_total_tokens = net_input_tokens + output_tokens + reasoning_output_tokens
cache_rate = cached_input_tokens / input_tokens
```

`net_*` and `cache_rate` are report-layer metrics and must not be written as new historical facts.

## 5. Architecture Constraints

The project has three modules:

- `token-meter-core`: shared app/collector configuration, Codex session parsing, team payload DTOs, common DTOs, and utilities.
- `token-meter-app`: dashboard/server entrypoint, HTTP/API, admin, report, SQLite CRUD/store, SQL resources, and static resources. Only app may depend on SQLite for server storage.
- `token-meter-collector`: teammate-side lightweight reporting entrypoint, Claude Code usage source, collector `.env` reading, and periodic upload. It must not start an HTTP server or include dashboard/admin/static resources.

Shared app/collector behavior belongs in core only when both sides truly need it.

## 6. Privacy Boundary

P5 must not collect, upload, store in report payloads, or log:

- prompt text
- response text
- raw JSONL
- raw API body
- raw transcript
- user input
- model output
- tool input payload
- tool output payload
- admin token
- device token plaintext
- device token hash
- complete local paths outside Local SQLite internal `source_files.path`

Local SQLite may store complete source paths in `source_files.path` for local incremental ingestion and issue diagnosis. Those paths must not enter canonical events, team payloads, reports, exports, stdout, or logs.
